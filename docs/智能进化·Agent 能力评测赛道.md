# 智能进化·Agent 能力评测赛道

## 赛题背景

随着大模型与数据处理技术的发展，Data Agent 正逐渐成为数据智能系统的核心形态。它的能力不再局限于单一模型推理，而是需要在复杂任务中完成规划、工具调用、多步骤执行与结果验证。

本赛道围绕真实产业中的海量数据处理与语料加工需求，鼓励参赛团队基于 MinerU 开发并提交能够解决实际场景问题、高效且高质量处理数据、并兼顾成本的数据智能体方案。

## 赛题任务

参赛团队需要开发并提交一个具备自动化数据处理能力的 Data Agent。系统需要能够在复杂任务环境中自动理解任务需求，调用工具或模块完成数据处理，生成结构化结果，并输出可验证的处理日志与结果。

系统需要支持通过 API 接口提交至组委会进行能力评测。

Data Agent 的处理对象主要面向文档或网页，例如 PDF、Word、PPT、HTML 等。系统应能系统性解决当前单一模型或传统数据处理管线面临的难点，包括但不限于：

- 财务报表中密集数字解析的准确性与防幻觉问题。
- 跨页合并与全局指代消解。
- 工程图、流程图或复杂图表解析。
- 低质量文档鲁棒处理，如严重模糊、光线不均、拍照件、手写签章重叠等。
- 基于规范或行业标准的解析与判定。

## 赛题能力要求

参赛团队设计和研发 Data Agent 时，需要在成本、速度和精度之间取得平衡，并满足以下三类能力。

### 任务一：数据理解与结构化处理

从多源数据中识别数据结构，完成信息抽取、格式转换、数据清洗和元数据抽取等操作，并输出统一规范的结构化数据结果。

### 任务二：复杂任务规划与自动执行

面对复杂任务目标后，能够自动拆解为多步子任务，合理调用工具完成处理流程。典型场景包括数据转换、批处理执行、任务调度、异常处理与恢复等。

### 任务三：系统稳定性与综合能力评测

系统需要在复杂或高负载条件下保持稳定执行，并确保结果输出的可靠性与一致性。该任务重点考察系统在模拟真实生产环境中的工程成熟度与可落地能力。

## 赛题硬性要求

- 必须使用 MinerU 工具链，包括 SaaS 端或开源项目。
- 需要提供可调用的 API 服务，便于组委会评测。
- 需要提供运行说明、接口说明、测试结果和可复现日志。
- 建议提供技术报告、演示材料和典型样例。

## 本项目对应解法

本项目选择财务报表核心指标抽取作为主攻场景，同时覆盖跨页指代、低质量 OCR 和行业规范审查等扩展场景。

核心思路：

1. 使用 MinerU 将 PDF、HTML 等文档解析为结构化 Markdown。
2. 使用 Agent 进行场景识别、上下文打包、规则抽取与 LLM 提取。
3. 对财务场景加入资产负债表恒等式校验，降低数字幻觉风险。
4. 通过 FastAPI 暴露异步任务接口，支持提交、轮询和健康检查。
5. 通过本地测试、并发稳定性测试和真实 PDF 联调验证系统可靠性。
