# Data Agent 技术报告

## 1. 项目概述

本项目面向“智能进化·Agent 能力评测赛道”，实现了一个基于 MinerU SaaS 与 DeepSeek 的 Data Agent，用于处理文档类数据，重点解决财务报表高密度数字抽取、跨页信息合并、低质量 OCR 文档去噪和结果一致性校验问题。

系统核心能力包括：
- 通过 MinerU 将 PDF/文档解析为结构化 Markdown。
- 通过 DeepSeek 对结构化上下文进行多轮提取与反思校验。
- 通过本地任务持久化支持异步提交、轮询查询和结果追溯。
- 通过健康检查和稳定性脚本验证服务可用性。

## 2. 整体设计方案

### 2.1 架构分层

- 接入层：FastAPI 服务提供 `/v1/agent/process`、`/v1/agent/task_status/{job_id}` 和 `/health`。
- 解析层：`mineru_client.py` 负责提交 MinerU 解析任务并轮询结果。
- 智能处理层：`agent.py` 负责 Markdown 抽取、场景路由、规则校验与 LLM 兜底。
- 状态层：`task_db.json` 保存任务生命周期，支持重启后继续查询。
- 验证层：`test_local_finance.py`、`run_real_pdf_test.py`、`test_stability.py` 分别覆盖本地烟测、真实联调与并发稳定性。

### 2.2 任务执行机制

1. 用户通过 API 提交任务，系统创建 `job_id` 并写入任务状态。
2. 后台任务启动后，调用 MinerU 提交解析请求。
3. MinerU 返回 `full_zip_url` 后，系统下载并解压 Markdown。
4. `agent.py` 按场景构建上下文，并优先执行规则化直接抽取。
5. 若规则抽取不足或不完整，再交给 DeepSeek 进行结构化提取。
6. 财务场景下进行一致性校验；必要时对缺失的 `所有者权益合计` 做恒等式补齐。
7. 最终结果写回本地任务库，供轮询查询。

### 2.3 数据处理与工具调用能力

- MinerU：负责把非结构化 PDF 转成可读 Markdown，降低版面错乱与表格碎行问题。
- DeepSeek：负责上下文理解、跨页联想、指代消解和 JSON 结果生成。
- 本地审计逻辑：负责财务平衡校验、字段缺失检测和异常重试。
- 日志输出：关键步骤均通过标准日志记录，便于排查和追溯。

## 3. 关键实现说明

### 3.1 财务抽取策略

财务场景优先采用直接抽取：
- 从 MinerU 生成的 Markdown 中定位财务报表章节。
- 按指标名称匹配对应数值。
- 跳过年份等误命中字段。
- 若 `资产总计`、`负债合计` 已存在但 `所有者权益合计` 缺失，则按恒等式补齐。

### 3.2 稳定性机制

- 使用 `task_db_lock` 保护任务状态写入。
- 任务状态落盘到 `task_db.json`，避免单纯内存态丢失。
- 过期任务会在启动和健康检查阶段清理。
- MinerU ZIP 下载采用重试和较长读超时。
- DeepSeek 调用设置超时，避免请求挂死。

## 5. 典型任务执行示例

### 示例 1：财务报表核心指标抽取
- 输入：中兴通讯 2020 年报 PDF。
- 场景：`financial`
- 输出：`营业收入`、`资产总计`、`负债合计`、`所有者权益合计`。
- 特点：直接抽取 + 规则校验通过。

### 示例 2：跨页合并与指代消解
- 输入：存在分页截断的说明性文档。
- 场景：`coreference_cross_page`
- 目标：恢复跨页断裂句子中的主语和代词指向。
- 特点：依赖上下文拼接和语言模型推理。

### 示例 3：低质量 OCR 鲁棒抽取
- 输入：模糊、遮挡、噪声较重的扫描件。
- 场景：`low_quality_ocr`
- 目标：结合上下文修复错别字和碎片化文本。
- 特点：强调容错与语义恢复。

### 示例 4：行业规范判定
- 输入：包含标准或合规条款的文档。
- 场景：`industry_spec`
- 目标：抽取规范要求并形成结构化判定结果。
- 特点：适合规则型审查任务。

### 示例 5：并发任务稳定性验证
- 输入：多个伪任务并发提交。
- 场景：内部测试。
- 目标：验证任务库不会串写、状态不会覆盖、过期任务可清理。
- 特点：覆盖工程成熟度与可用性。

## 6. 系统适用场景与应用价值

适用场景：
- 财务报表抽取与审计辅助。
- PDF/文档结构化数据生产。
- 低质量文档修复与文本恢复。
- 规则审查与行业标准判定。

应用价值：
- 降低人工整理高密度文档的成本。
- 提高结构化抽取的准确率和一致性。
- 提供可追溯的任务状态和日志记录。
- 便于扩展到更多文档理解场景。
